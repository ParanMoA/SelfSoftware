- git & git hub

→ git은 버전 관리 ‘프로그램’

→ git hub는 버전 관리, 소스 코드 공유, 분산 버전 제어 등등이 가능한 원격 ‘저장소’

- git bash

→ cmd(명령 프롬프트)같은 것

→ 윈도우용 git은 리눅스와 동일한 명령어를 실행할 수 있도록Git Bash를 제공한다

→ 리눅스나 Mac에서 터미널을 사용하는 것처럼 윈도우에서도 리눅스와 동일한 명령어를 실행할 수 있도록 한다.

→ 윈도우에서 리눅스 명령어 사용해서 git작업이 가능 

- 체크섬이란?

→ 데이터의 모든 수치를 더한 값

→ 주로 데이터 전송 시 전송된 데이터의 변조, 해킹, 오류 또는 파손 등을 검사하는데 사용

→ 데이터 전송 시 데이터와 데이터의 합(체크섬)을 같이 전송하고, 받는 쪽에서 다시 데이터를 합산해서 전송된 합(체크섬)과 같은지 비교함으로써 데이터 파손 여부를 검사

- Git의 무결성

**→** Git은 데이터를 저장하기전에 항상 체크섬을 구하고 체크섬으로 데이터 관리

→ ****체크섬을 이해하는 Git 없이는 어떠한 파일이나 디렉토리도 변경 불가

→ ****체크섬은 Git에서 사용하는 가장 기본적인 데이터 단위이자 Git의 기본철학

→ ****Git은 SHA-1 해시를 사용하여 체크섬을 만듬

→ ****실제로 Git은 파일을 이름으로 저장하지 않고 해당 파일의 해시로 저장

→ 각 커밋은 이전 커밋의 chekshum(해시값)을 가지고 있다.

- 리모트 트래킹 브랜치란?

→ 리모트 브랜치를 추적하는 브랜치

→ 리모트 트래킹 브랜치는 리모트 서버에 연결할 때마다 리모트 브랜치에 따라서 자동으로 움직임

→ 리모트 저장소의 마지막으로 연결했던 순간에 리모트 브랜치가 무슨 커밋을 가리키고 있었는지를 나타냄
